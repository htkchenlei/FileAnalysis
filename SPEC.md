# 文件分析系统规格说明

## 1. 项目概述

### 项目名称
招标文件智能分析系统 (FileAnalysis)

### 项目类型
文档处理工具类 Web 应用

### 核心功能概述
用户上传招标文件（PDF、Word 等格式），系统调用 markitdown 工具将文档转换为 Markdown 格式，并在网页上展示分析结果。

### 目标用户
- 需要分析招标文件的企业用户
- 采购人员、投标专员
- 法务人员、财务人员

## 3. 功能规格

### 3.1 核心功能

#### 文件上传
- 支持格式: PDF, DOCX, DOC, TXT, RTF
- 文件大小限制: 50MB
- 拖拽上传 + 点击选择两种方式
- 上传进度条显示

#### 文档转换
- 调用 markitdown 库进行格式转换
- 转换过程中显示加载动画
- 转换失败时显示错误信息

#### 结果展示
- Markdown 渲染为 HTML
- 支持代码高亮
- 支持表格渲染
- 支持图片展示

#### 历史记录
- 保存每次转换的记录
- 记录内容包括: 文件名、转换时间、状态
- 支持查看历史结果
- 支持删除历史记录

### 3.2 用户交互流程

```
用户访问首页 → 上传文件 → 系统处理 → 展示结果 → 保存历史
     ↓
  查看历史记录 → 选择记录 → 查看结果
```

### 3.3 数据模型

#### 文件记录表 (files)
| 字段 | 类型 | 描述 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| filename | TEXT | 原始文件名 |
| original_name | TEXT | 用户上传时的文件名 |
| content | TEXT | 转换后的 Markdown 内容 |
| status | TEXT | 状态: pending, processing, completed, failed |
| error_message | TEXT | 错误信息 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3.4 API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/upload | 上传文件并转换 |
| GET | /api/files | 获取文件列表 |
| GET | /api/files/:id | 获取单个文件详情 |
| DELETE | /api/files/:id | 删除文件记录 |

## 4. 技术架构

### 4.1 前端
- **框架**: Vue 3 (Composition API)
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **样式**: Tailwind CSS 3
- **Markdown 渲染**: markdown-it
- **HTTP 客户端**: Axios
- **构建工具**: Vite

### 4.2 后端
- **框架**: Flask
- **数据库**: SQLite + SQLAlchemy
- **文件处理**: markitdown
- **CORS**: Flask-CORS
- **文件存储**: 本地文件系统

## 5. 验收标准

### 5.1 功能验收
- [ ] 用户可以上传 PDF、DOCX 等格式文件
- [ ] 上传后系统自动调用 markitdown 转换
- [ ] 转换结果以 Markdown 形式展示
- [ ] 用户可以查看历史记录
- [ ] 用户可以删除历史记录
